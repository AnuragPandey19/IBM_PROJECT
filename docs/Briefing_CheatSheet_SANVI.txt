--------------------------------------------------------------------
S20 — MULTI-AGENT TEST CASE GENERATION
--------------------------------------------------------------------
Say (open with the story):
  "Our ground truth is not authored by one person or generated by one
   model — it is orchestrated across 12 diverse LLM agents in a
   producer + validator pattern."

Then walk the 6 cards (short — 15 seconds each):

  1. Per-member pipeline —
     "Three LLM agents generate test cases in parallel, independent
      of each other. A fourth agent validates."

  2. Validator — fix or regenerate —
     "The validator checks scenario-label consistency, value
      plausibility, feature schema, and JSONL row shape. Fixable
      errors are auto-corrected. Semantically wrong rows go back to
      the originating agent to regenerate."

  3. Provider diversity —
     "Each generator uses a different API provider — Claude, GPT,
      Grok, DeepSeek, NVIDIA, and others. No single model's blind
      spot leaks into our ground truth."

  4. Team scaling —
     "3 team members × 4 agents = 12 agents active simultaneously.
      Every batch is composed of a different API mix, so authorship
      is genuinely diverse."

  5. Manual quality gate —
     "The aggregated 12-agent output lands with the project lead
      for a manual pass — sanity-check labels, spot drift, resolve
      conflicts. Only after this gate does the batch enter the
      Sparkov scoring runner."

  6. Model isolation guarantees —
     "Only the 8 checkout fields reach the model — labels are held
      back. Test cases are never stored in the model or trained on —
      zero fine-tuning. Labels and typology metadata are used only
      for post-scoring comparison."

Bottom stats to remember:
  - 12 diverse LLM agents active
  - V1 · 714  combined labeled set
  - V2 · 150,000  large-scale stress set
  - 150,714 total labeled rows

Handoff:  "The visual for this pipeline is on the next slide."

S22 — FINAL TESTING RESULTS (V1 + V2)
--------------------------------------------------------------------
Say (walk both cards):

V1 · Combined labeled set — 714 rows, 341 fraud, 373 legit:
  - 437 correct, 245 sent to review, 32 wrong
  - Accuracy (excluding review): 93.2%
  - Fraud rows: 125 blocked, 189 review, 27 approved
  - Legit rows: 5 blocked, 56 review, 312 approved
  - 0.6% false-block on legit — customer-friendly
  - 7.9% missed fraud; most fraud lands in review (55%) —
    reviewable, not lost

V2 · Large-scale stress set — 150,000 rows, 62,119 fraud, 87,881 legit:
  - 100,050 correct, 42,854 sent to review, 7,096 wrong
  - Accuracy (excluding review): 93.4%
  - Fraud rows: 27,292 blocked, 30,697 review, 4,130 approved
  - Legit rows: 2,966 blocked, 12,157 review, 72,758 approved
  - 3.4% false-block at scale — production-acceptable
  - 6.6% missed fraud; another 49% land in review

Close with the pattern:
  "Consistent result across two independently-authored test sets —
   ~93% accuracy on decisive predictions, near-zero false-block,
   and a controlled review bucket."
